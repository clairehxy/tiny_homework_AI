# AI课程作业 - 思维树(Tree of Thoughts)应用

## 题目1：哪些情况下适合使用思维树(Tree of Thoughts, ToT)?

思维树(ToT)适用于以下场景：

**1. 需要多步规划与决策的复杂任务**
当问题无法通过单一路径直接求解，而需要在多个中间步骤中进行探索时，ToT非常适用。例如"24点游戏"，需要从多个数字组合和运算顺序中搜索可行解，ToT通过构建思维树探索不同运算路径，显著提升了求解成功率（从4%提升至74%）。

**2. 存在多种可能解的创意生成任务**
在创意写作、方案设计等开放性任务中，ToT可以生成多个不同的思维分支（如不同的故事走向、设计方案），并通过评估选择最优路径。例如写作时，ToT可探索多个情节发展方向，评估哪个更具逻辑性和吸引力。

**3. 需要回溯和修正的约束满足问题**
对于数独、填字游戏、华容道等约束满足问题，ToT允许模型在发现某条路径无效时回溯到上一节点，尝试其他分支。这种"试错-评估-回溯"机制类似于人类的解题策略，比线性思维链(CoT)更具灵活性。

**4. 资源分配与调度优化问题**
在会议安排、任务调度、路径规划等组合优化问题中，ToT可以系统地探索不同的分配方案，评估每种方案是否满足约束条件（如时间冲突、容量限制），最终找到全局最优解。

**总结**：ToT的核心优势在于**分支探索**和**全局评估**，适合需要搜索、规划和决策的复杂任务，而不适合简单的问答或事实检索任务。

## 题目2c：会议安排工具

### 项目简介

本项目使用Python语言和思维树(ToT)方法实现了一个智能会议安排工具。该工具能够自动为多个会议分配会议室和时间槽，同时满足以下约束条件：

- **时间不冲突**：同一参会人员在同一时间只能参加一个会议
- **房间容量匹配**：会议室容量必须大于等于参会人数
- **会议室不冲突**：同一会议室在同一时间只能安排一个会议
- **设备需求**：会议室必须配备会议所需的设备（投影仪、白板等）
- **优先级处理**：高优先级会议优先安排

### 思维树(ToT)实现方法

本项目的核心算法采用思维树方法，具体实现如下：

**1. 节点设计（思维状态）**
每个节点代表一个部分安排方案，包含：
- 已安排的会议列表（含时间、地点分配）
- 剩余待安排的会议列表
- 当前资源占用状态（人员时间表、会议室时间表）

**2. 分支生成（思维扩展）**
对于每个待安排会议，生成所有可能的分配方案分支：
- 遍历所有可用的时间槽
- 遍历所有满足容量和设备需求的会议室
- 检查是否与已安排会议冲突（人员、地点）
- 每个有效组合生成一个新的子节点

**3. 评估函数（思维评估）**
对每个节点进行评估打分：
- 基础分：成功安排的会议数量
- 加分项：高优先级会议优先安排、会议室利用率合理
- 减分项：时间碎片化、重要人员时间冲突

**4. 搜索策略**
采用束搜索(Beam Search)策略：
- 每层保留评分最高的K个节点（默认K=3）
- 优先扩展高分节点，剪枝低分路径
- 最终选择得分最高的完整方案

**5. 回溯机制**
当某条路径无法完成所有会议安排时，自动回溯到父节点，尝试其他分支。这种机制确保在复杂约束下仍能找到可行解或近似最优解。

### 使用方法

**1. 环境要求**
- Python 3.8+
- 无需额外依赖（仅使用标准库）

**2. 运行示例**
```bash

```

**3. 自定义输入**
在 `examples.py` 中修改或添加会议、会议室、人员数据，然后运行主程序。

**4. 输出结果**
程序将输出：

- 未能安排的会议及原因

### 代码示例

```python
from tot_scheduler import ToTScheduler
from models import Meeting, Room, Person

# 创建调度器
scheduler = ToTScheduler(beam_width=3, max_depth=10)

# 添加会议室
scheduler.add_room(Room(id="R1", name="会议室A", capacity=20, equipment=["projector", "whiteboard"]))

# 添加人员
scheduler.add_person(Person(id="P1", name="张三", role="manager"))

# 添加会议
scheduler.add_meeting(Meeting(
    id="M1", 
    name="项目评审", 
    duration=60,  # 分钟
    attendees=["P1", "P2"],
    required_equipment=["projector"],
    priority=3  # 1-5，数字越大优先级越高
))

# 执行调度
schedule = scheduler.schedule()
print(schedule)
```

## 题目2a和2b的解题思路

### a. 求解华容道小游戏

**解题思路：**
华容道是一个经典的滑块拼图游戏，目标是通过移动棋子将特定棋子（曹操）移动到出口位置。

**ToT方法应用：**
1. **状态表示**：每个节点表示棋盘的一个布局状态（各棋子位置）
2. **分支生成**：对每个状态，生成所有合法的移动操作（上下左右移动可移动的棋子）
3. **评估函数**：
   - 曹操到出口的距离（曼哈顿距离）
   - 阻挡曹操路径的棋子数量
   - 已移动步数（避免循环）
4. **搜索策略**：使用A*算法或束搜索，优先扩展评估分高的节点
5. **剪枝**：记录已访问状态，避免重复搜索

**关键挑战**：状态空间巨大（约20万种布局），需要高效的评估函数和剪枝策略。

### b. 求解数独

**解题思路：**
数独是在9×9格子中填入1-9数字，满足每行、每列、每个3×3宫格内数字不重复。

**ToT方法应用：**
1. **状态表示**：每个节点表示一个部分填充的数独网格
2. **分支生成**：选择空格最少（约束最强）的格子，尝试所有可能的数字
3. **评估函数**：
   - 已填充格子数量
   - 剩余空格的可选数字数量（越少越好，说明约束强）
4. **约束传播**：在每次填数后，立即排除同行/列/宫格中的相同数字，减少分支
5. **回溯**：当某个格子无可用数字时，回溯到上一步

**关键优化**：
- 使用最小剩余值(MRV)启发式选择下一个填充的格子
- 前向检查(Forward Checking)提前发现冲突
- 相比暴力搜索，ToT的评估和剪枝可大幅减少搜索空间
